On this day, one year ago, I announced the first version of Dungeon Mercenary on brogue’s reddit. In this post, I’ll discuss how this year went.

Statistics about this first year

I’ve released the game for almost a year on:

  • itch.io where it gathered 2574 views (which includes loading the HTML5 version) of the game, 153 downloads (desktop version of the game), and one 1€ donation (meaning after paypal and itch, I earned 60 cents).
  • GameJolt where it gathered 3100 views and 1100 plays (I don’t know if this is solely HTML5 or if it counts downloads too). The game had a strong start on GameJolt as any new game is featured on GameJolt’s front page during a week. During this week I already got more than 1000 views. If I had known that beforehand, maybe I would have waited a bit more to release a more polished game. I have 0.91€ balance, thanks to the ads that GameJolt serve.

This means I made 1,50€ in one year with Dungeon Mercenary. Not even a coffee in Paris Gare de l’Est… Tomorrow’s not the day I’ll quit my day job to become an indie developer (not that it’s a surprise though ^^).

I almost never had any feedback from itch, neither from itch’s developers, nor from players on itch. On GameJolt though, the game got ranked 51 times (for an average of 3,6 stars (out of 5)), gathered 51 followers, I’ve had a number of exchanges with players, and the game was selected by the website for a competition where it ended up something like 16/75 (not bad for an ASCII game I believe). I was affected once by a bug on GameJolt’s website, and it got immediately fixed after I notified them; so kuddos GameJolt.

Thoughts on this year

  • I learnt quite a lot on libgdx, the Java library I rely upon for display. I learnt about the Google Web Toolkit library (which allows me to release to HTML5). Both are great tools.
  • Being able to ship to a lot of platform (but especially HTML5) is important. Having a standalone .exe for Windows players is important. No one will install anything to run your game, no matter how fancy it is.
  • It is very hard to get feedback and build a community. Only 3 players consistently give me feedback/write a comment/tell something after a new release. Maybe I’m not good at attracting players. Generally speaking, twitter seems like a good tool communicate, as they are a lot of people there. For more roguelike-focused people, reddit’s roguelikedev and roguelike seem like the place where there are many people, oh and also roguebasin. Every time I release, I see hits on my webpage coming from roguebasin.
  • No one ever did a let’s play or a review/post about Dungeon Mercenary, except for one guy that was in a retro-gaming challenge, for the “a retro game in 2016” entry.
  • Being a dad and neither a slacker nor at home nor at work means it is tremendously difficult to progress. Every release is a miracle, because I need like a big hour straight to build it, upload it and announce it everywhere.

The year to come

Despite that creating an ASCII indie game is a very solitary experience, I’m pumped to continue, because my brain’s just throwing ideas continuously; and I’m not seeing that as a “one game” experience. My design is usually pretty generic, so I’m not feeling as “developing Dungeon Mercenary”, I’m feeling as “developing a framework for 2D grid games” which I’ll be able to instantiate to bring my ideas to life!

So talk to you again soon!
