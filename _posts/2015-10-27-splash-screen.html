Last week, I wrote the splash screen:

There is no support for buttons in SquidLib, but it was pretty easy to program a generic panel displaying buttons vertically, aligning on the grid’s cells. For margins, I used libgdx ShapeRenderer to draw 4 rectangles and filled arcs. I’ll soon put in SquidLib this generic implementation, offering automatic layout of buttons, provided their text and some measures.

The implementation is pretty nice to the client, as it computes the mouse listener to handle clicks on buttons and plug a custom key listener that reacts to the pressing of the highlighted characters (‘n’, ‘c’, and ‘q’ above).

The implementation offers three styles for the corners, the screenshot above shows the Rounded style, there’s a rectangular style, and a Missing style that has a NES-flavor (to my taste ^^):

Below is the code to build this screen. It is an abstract class, with a single method to fill: selectedButton(int) that is called when a button is selected.

final List<IColoredString<Color>> buttonsText = ...;
final SquidPanel bg = buildScreenWideSquidPanel(nbVCells);
final SquidPanel fg = buildScreenWideSquidPanel(nbVCells);
buttonsPanel = new ButtonsPanel<Color>(bg, fg, buttonsText, colorCenter.getButtonsInside(), colorCenter.getButtonsMargin()) {
    @Override
    protected void selectedButton(int i) {
        if (i == 0) {
          /* 'New Game' was selected */
          ...
        } else if (i == 1) {
          /* 'Credits' was selected */
          ...
        }
        ...
    }
};
buttonsPanel.setInterButtonMargin(1);
buttonsPanel.setXPadding(2);
buttonsPanel.setMargin(10);
buttonsPanel.setShortcutCharacterColor(colorCenter.getShortcutColor());
buttonsPanel.setBackgroundColor(colorCenter.getDefaultBackgroundColor());
stage.addActor(buttonsPanel);
/* This builds the panel and return the InputProcessor */
final InputProcessor pi = buttonsPanel.putAll();
Gdx.input.setInputProcessor(pi);
stage.draw();
